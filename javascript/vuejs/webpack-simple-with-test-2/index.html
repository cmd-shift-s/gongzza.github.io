Webpack2로 Vue.js 테스트 환경 구축하기 - 2

2017-01-21
2021-02-21
javascript, mocha, should, test, vue.js, webpack2

이번에는 이전 포스팅 Webpack2로 Vue.js 테스트 환경 구축하기 - 1에서 말했던 것처럼 test/unit/util.js를 추가해서 테스트 코드를 좀 더 쉽게 만들 생각이다.

먼저 뷰 컴포넌트 생성 부분을 메소드로 빼고 $el을 jQuery로 감쌀 생각이다.

$el은 HTMLElement이라서 DOM접근에 최적화 되어있는 jquery를 사용하면 원하는 element에 쉽게 접근할 수 있다.
특히 리스트 형식일 경우 처음 리스트 갯수를 검사하고 데이터를 검사하기 위해서 만들어야 하는 코드가 많이 간결해 졌다.

다음은 jquery를 사용해서 만들어본 테스트이다.
$el은 HTMLElement이고, $은 jquery의 검색 결과를 리턴한다.

jquery를 돔접근 용도로 사용한다면, 테스트코드 가독성이 훨씬 올라갈 것이라고 생각한다.

Test Environment

util.js를 추가해서 테스트 코드를 간결하게 만들어 보자.

먼저 jquery와 babel-preset-stage-0을 설치한다.

.babelrc에 stage-0를 추가하고 모듈을 사용하기 때문에 modules: false를 제거한다.

util.js를 작성한다.

Test

이제 App.spec.js를 util.js를 이용하도록 수정해 보자.

테스트를 돌려보면 통과하는 것을 확인할 수 있다.

propsData

vm.msg를 propsData로 넘겨서 데이터가 잘 적용이 되는지 검사하는 테스트를 작성해 보자.

App.vue도 msg를 props로 받도록 수정한다.

테스트를 확인해보면 잘 통과하는 것을 볼 수 있다.

이렇게 프론트 엔드에서 테스트가 힘들었던 데이터와 화면검증 테스트를 할 수 있게 되었다.

다음은 Faker.js를 이용한 mock data와 axios를 이용한 비동기 초기화시 axios-mock-adapter를 이용한 fake request를 만들어서 서버 없이 통신 테스트를 할 수 있도록 만들어 볼 것이다.
