Webpack2로 Vue.js 테스트 환경 구축하기 - 1

2017-01-21
2021-02-21
javascript, mocha, mocha-loader, should, test, vue.js, webpack2

vue-template인 webpack-simple를 기반으로 테스트 환경을 구축해 나갈 것이다.

webpack-simple을 선택한 이유는 webpack2를 사용하는 것과
정말 필요한 Babel과 Vue.js만 설치해주기 때문이다.

먼저 vue-cli를 글로버로 설치해줘야 한다.

Install

vue-cli를 이용해서 프로젝트를 생성한다.

생성된 디렉토리에 들어가서 페키지를 설치하고 실행해 보자.

위 메시지가 나오면서 시스템에서 기본 프로그램으로 지정한 웹브라우저가 열리면서 아래의 화면이 나오면 정상적으로 설치가 완료된 것이다.

hot모드 이기 때문에 소스를 수정하면 화면 재시작 없이 바로 적용이 된다.

ESLint를 적용하는 방법은 webpack2에 eslint 붙이기를 참고하기 바란다.

Test modules

이제 테스트에 필요한 것들을 설치할 것이다.

나는 mocha.js, Should.js, Sinon.JS을 이용할 것이다.

sinon은 spy를 이용해서 Vue component의 메소드가 호출 되어있는지 확인할 때 사용한다.

그리고 webpack에서 돌리기 위해서 mocha-loader를 추가할 것이다.

이제 테스트에 필요한 모듈들이 갖춰졌다.

Test Environment

테스트 환경을 구축해보자.

먼저 webpack.config.js에 test를 추가한다.
alias도 추가해주자. 그래야 test에서 접근하기가 편하다.

그리고 index.html를 수정하고 test.html파일을 생성하자.

mocha-loader가 호출하는 코드를 작성한다.
여기서 테스트들을 호출한다.

나는 주로 $ npm run dev스크립트에서 --open옵션을 제거한다.
이유는 에러가 낫거나 새로운 파일을 추가 했을 경우 서버를 재시작 하는 경우가 많은데,
매번 웹 페이지가 새로 열리는 것이 번거롭기 때문이다.

Test

이제 테스트 스팩을 만들어 보자.

먼저 간단하게 App.vue의 msg가 원하는 위치에 출력이 되었는지 검사해 보자.

서버를 실행 시키고 http://localhost:8080/test.html을 접속해보면 테스트가 통과된 것을 확인할 수 있다.

여기까지가 가장 기본 적인 테스트 환경일 것이다.

다음에는 test/unit/util.js를 만들어서 좀 더 간결하게 테스트코드를 작성할 수 있는 방법을 알아보겠다.
