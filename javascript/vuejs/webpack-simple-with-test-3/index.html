Axios

이제 axios를 사용해 보자.

다음과 같은 동작을 검증할 것이다.

  1. App.vue에서 /api/me를 호출한다.
  2. /api/me에서 내려준 데이터가 App.vue에서 재대로 대입되었는지 확인한다.
  3. /api/me에서 받은 데이터중 username데이터가 .username에 내용에 반영이 되었는지 확인한다.

createAxiosMock은 단순 생성 코드만 있지만, 테스트코드에서 axios와 axios-mock-adapter를 import하지 않아도 되서 코드양이 줄어든다.

테스트 코드를 작성한다.

axiosMock에서 GET메소드로 요청되는 /api/me를 받으면 상태값 200에 faker로 생성된 user mock data를 내려준다.

이제 App.vue를 작성한다.

테스트 코드를 돌려보면 실패를 한다.

에러가 나온 이유는 vm.user.username가 undefined인데 should을 접근을 하려고 해서 참조할 수 없어서 에러가 발생한 것이다. 즉, vm.user가 셋팅이 되기전에 참조를 한 것이다.

이유는 테스트 코드를 보면 비동기가 아닌 동기방식으로 테스트를 하고 있다. 하지만 axios는 Promise방식으로 비동기로 데이터를 호출하고 있다. 그래서 데이터를 받고 셋팅하기 전에 테스트를 수행해 버린 것이다.

이 문제를 해결하기 위해서 Promise를 이용하였는데,
먼저 process.nextTick으로 axios비동기 이벤트 다음으로 처리하도록 하였고,
vm.$nextTick을 두번 호출해서 updated가 되었을 때 vm을 리턴하도록 하였다.
vm.$nextTick을 두번 호출한 이유는 Vue.js#Lifecycle에서 보면 알수 있듯이 beforeUpdate와 updated를 기다리기 위해서이다.

asyncNewVM함수를 이용해서 테스트 코드를 다시 작성해 보자.

테스트 콜백함수에 done파라메터를 추가해서 async 테스트로 만든다.

vm 생성을 Promise로 받아서 테스트를 수행한다.
테스트가 실패했을 경우 catch로 받아서 done로 넘겨줘서 테스트가 실패했음을 알린다.

이정도 테스트 환경이면 왠만한 테스트는 모두 가능할 것이다.

여기서 다음으로 해야 할 것은 src 디렉토리를 보면서 변경이 일어날 때마다 테스트를 수행하는 것이다.
