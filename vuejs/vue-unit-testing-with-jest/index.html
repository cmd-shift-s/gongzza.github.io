Vue.js Unit Testing with Jest

2017-11-05
2021-02-21
jest, vue-test-utils, vuejs

Jest를 이용해서 Vue.js 테스트 환경을 만들어 볼 것입니다.

Test Runner로 Mocha + Webpack도 있었지만, Jest를 선택하게 되었습니다.

  1. 테스트 실행이 굉장히 빠릅니다.
  2. 스냅샷을 찍어서 비교가 가능합니다. 이는 직접 DOM이 어떻게 그려질지 확인할 수 있습니다.
  3. Assertion(Expect)과 Code Coverage 등의 기능들이 내장되어 있습니다.
  4. 요즘 대새인 Zero Configuration
  5. 다양한 CLI 옵션들이 있습니다.

좀 더 고민이 되는 경우 Test Runner 선택하기를 참고하시면 도움이 될 것 같습니다.

시작하기

Testing Single File Components with Jest에서 설명이 매우 잘 되어 있습니다.

그래서 위 내용을 참고로 작성 하였습니다.

프로젝트 생성

vue-cli를 이용해서 webpack-simple 탬플릿으로 프로젝트를 생성하겠습니다.

Jest 셋팅

Jest와 vue-test-utils를 설치합니다.

그리고 *.vue 파일을 처리하기 위해서 vue-jest를 설치합니다.

Package.json

package.json에 Jest 설정 옵션을 정의합니다.

Handling webpack aliases

webpack config에서 resolve alias를 사용하고 있다면, Jest 에서도 설정해 주어야 합니다.

Configuring Babel for Jest

babel을 사용하고 있다면, babel-jest도 설치가 필요합니다.

기본적으로 babel-jest는 자동으로 설치된다고 합니다. 하지만, 우리가 *.vue를 추가 하면서 babel-jest을 명시적으로 추가해 주어야 합니다.

그리고 .babelrc에 env.test를 설정합니다.

async/await을 사용할 경우 plugins에 transform-runtime를 추가합니다.

Snapshot Testing

vue-server-renderer를 사용해서 Jest snapshot testing을 수해할 수 있습니다.

그리고 package.json에 설정합니다.

Test 예제

기본으로 설치된 src/App.vue에서 h1 태그에 msg 값이 잘 설정되는지 확인해보는 테스트를 작성해 보겠습니다.

테스트를 실행해 봅니다.

테스트가 성공적으로 통과하는 것을 확인할 수 있습니다.

그리고 test/__snapshots__/app.test.js.snap 파일이 생성된 것을 볼 수 있습니다.
(스냅샷 파일이 없을 경우 자동으로 생성됩니다.)

생성된 파일을 확인해 봅시다.

앞으로 테스트 할 때 랜더링된 정보가 위 내용과 다를 경우 테스트가 실패하게 됩니다.

src/App.vue에서 <img src="./assets/logo.png">를 제거한 다음 테스트를 다시 수행해 봅니다.

정확하게 이전 스냅샷과의 차이점을 보여주고 있습니다.

만약 화면이 수정되어서 스냅샷이 업데이트 되어야 할 경우 npm test -- -u 명령어로 스냅샷을 업데이트 해주면 됩니다.

만약 vue-server-renderer를 사용할 경우 this.$el.classList 같은 메소드에 대해서 재대로 랜더링이 되지 않는 문제가 있습니다.

Code Coverage

위에서 언급한데로 Jest는 Code Covarge 기능도 갖추고 있습니다.

CLI로 손쉽게 사용할 수 있습니다.

함수형 컴포넌트는 다음 설정을 추가해야 Code Coverage를 확인할 수 있습니다.

결론

Jest의 스냅샷 기능으로 어느정도 화면단의 사이드 이팩트를 확인할 수 있는 좋은 수단이 된 것 같습니다.

테스트 러너 별로 예제도 잘 되어 있고 설명도 잘 되어 있으니 마음에 드는 걸로 테스트를 적용해 보시기 바랍니다.
